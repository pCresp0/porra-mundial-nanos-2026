    ¿Cómo está hecha esta web?

    Un dashboard hecho a mano para seguir la porra del Mundial 2026. Sin frameworks de frontend, sin base de datos clásica y con automatización completa de resultados vía GitHub Actions. Todo el código está abierto en GitHub.

    🗂️ Arquitectura general

    La web tiene dos modos de ejecución:

    • Local (dev) — un servidor Flask (Python) lee el Excel de pronósticos con openpyxl, construye el data.json en memoria y lo sirve en localhost:5050.
    • Producción — GitHub Pages sirve ficheros estáticos: index.html + data.json pregenerado. No hay servidor en producción.

    El frontend es una SPA de una sola página. Todas las pestañas se renderizan con JS puro; no hay React, Vue ni nada similar.

    ⚙️ Automatización de resultados

    Un workflow de GitHub Actions se ejecuta cada 5 minutos:

    • should_update.py — decide si vale la pena actualizar (ventana de ±2 min antes del partido hasta 6 h después). Si no hay partidos cercanos, salta el resto.
    • fetch_results.py — llama a la API de resultados, parsea goles (incluyendo OG y penaltis), escribe data/scorers.json y data/live.json.
    • excel_sync.py — vuelca los resultados al Excel del repositorio.
    • build_static.py — regenera data.json y hace git push. GitHub Pages publica automáticamente.
    • fetch_highlights.py — busca resúmenes de DAZN en YouTube (API v3) para cada partido jugado y los inyecta en data.json.

    Además, hay un backup nocturno (rama diaria) y un snapshot de visitas cada hora.

    📡 API de resultados

    Se usa la API pública worldcup26.ir/get/games (104 partidos). Los campos clave son time_elapsed (minuto en juego o "finished"/"notstarted"), home_score / away_score y las cadenas de goleadores con formato "Nombre MM'(OG)" / "Nombre MM'(P)".

    El parser en _parse_scorers() extrae jugador, minuto (incluyendo tiempo añadido como 45'+3'), y flags de gol en propia y penalti con regex.

    📊 Cálculo de puntos (en tiempo real)

    La función _score_breakdown() en app.py compara el pronóstico de cada jugador con el resultado real:

    • Signo 1X2 — 1 pt si aciertas victoria local, empate o victoria visitante.
    • Diferencia — 2 pts adicionales si la diferencia de goles coincide.
    • Resultado exacto — 3 pts adicionales si el marcador es exacto (total 6 pts).

    Durante un partido en juego, data/live.json alimenta los puntos provisionales que se muestran en la clasificación con el badge animado.

    🗃️ Ficheros clave

    • app.py — servidor Flask + toda la lógica de negocio (build_data()).
    • fetch_results.py — cliente de la API + parser de goles.
    • fetch_highlights.py — busca resúmenes DAZN en YouTube (API v3) con rate-limiting.
    • fixture_data.py — fixture completo: 104 partidos, sedes, horarios ES, TV.
    • team_players.py — jugadores destacados por selección + ficha individual (TheSportsDB).
    • excel_sync.py — lectura/escritura del Excel de pronósticos.
    • build_static.py — genera data.json estático para producción.
    • should_update.py — guardia del workflow: evita llamadas innecesarias a la API.
    • static/js/app.js — todo el frontend (~8 500 líneas de JS vanilla).
    • data/live.json — partidos en juego ahora mismo (vacío cuando no hay partido).
    • data/scorers.json — goleadores de todos los partidos finalizados.
    • data/highlights.json — IDs de vídeos de resúmenes encontrados en YouTube.

    🛠️ Stack técnico

    Frontend

    HTML5 CSS (vars + keyframes) JS vanilla ES2020 Tailwind CSS (CDN) Chart.js Leaflet Bebas Neue

    Backend / scripts

    Python 3.12 Flask 3.x openpyxl urllib (stdlib) hashlib (SHA-256)

    APIs externas

    worldcup26.ir (resultados) YouTube Data API v3 (resúmenes) TheSportsDB (fichas jugador)

    Infraestructura

    GitHub Pages GitHub Actions (cron 5 min)
